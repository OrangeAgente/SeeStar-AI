# Scattered-Light / Halo Metric (Tier-2 QA) — Design Spec

**Date:** 2026-07-05
**Project:** seestar-mcp
**Status:** Approved direction (from discussion) — ready for implementation planning
**Builds on:** Phase 1 Tier-2 QA (`qa_tier2.py`).

---

## Goal

Catch **thin high cirrus / scattered light** that the current metrics can miss. Today Tier-2 grades on FWHM, HFR, eccentricity, SNR, background *level*, star count, and wFWHM. A faint, roughly uniform veil that only mildly lifts the background — putting **halos around bright stars and softening contrast** — can stay under the session-relative SNR and star-count floors and be graded PASS. Add a dedicated **scattered-light metric** that measures the *signature* of a veil (bright-star halos + large-scale background structure), so those subs get flagged instead of quietly softening a stack.

## Why the existing metrics miss it

- **Background level** is a single sigma-clipped median — a smooth veil raises it uniformly, but the *session-relative* threshold shifts with it, so a whole-session haze doesn't stand out.
- **SNR / star count** catch *significant* transparency loss, but a thin veil can dim faint stars only slightly and still clear the floors.
- Nothing today looks at **local background structure** (halos around bright stars) or **large-scale background non-uniformity** (gradients) — which is exactly what scattered light produces.

## Design principles (inherited)

Deterministic (pure function of the FITS), never-raise (a bad sub returns `None` for the metric, never crashes), reason-tagged verdicts, session-relative thresholds with absolute overrides, no new heavy deps (photutils/astropy/numpy already pinned). Backward compatible: absent/None metric must not change existing verdicts.

---

## Architecture

Extend `src/seestar_mcp/planning/../qa_tier2.py` (the existing Tier-2 module) with one new sub-metric + one verdict rule; add two config thresholds; add a synthetic `hazy` fixture. No new module.

### The metric — two complementary components → one `scattered_light` figure

Computed on the same detected sources / background the analyzer already produces:

1. **Bright-star halo ratio.** For the brightest `K` (default 10) detected stars, measure the sky in a *halo annulus* just outside the stellar core (inner radius ≈ a few × FWHM, outer a bit larger) via `photutils.aperture.CircularAnnulus` + `aperture_photometry`. The **halo ratio** for a star = `(annulus_median − global_background) / (star_peak − global_background)`. A star sitting on a raised local pedestal (a veil-scattered halo) has an elevated ratio. Take the median over the K brightest stars. Clean sky → ~0; cirrus → elevated.
2. **Background non-uniformity.** From a coarse `photutils.background.Background2D` mesh, compute the spread of the background across the frame relative to its median: `bkg_nonuniformity = (mesh.std) / (median + eps)` (or max−min normalized). Scattered light and gradients raise this; a flat dark sky is ~0.

Combine into a single **`scattered_light`** scalar (document the blend, e.g. a weighted sum or max of the two normalized components, clamped ≥ 0). Store on `SubMetrics` as `scattered_light: float | None` (None if it can't be computed — too few stars / load failure — so it never forces a verdict).

### Verdict rule (session-relative, additive)

- New config on `Settings`: `qa_scatter_reject_sigma: float = 2.0`, `qa_scatter_marginal_sigma: float = 1.0`, and optional absolutes `qa_scatter_absolute: float | None = None`.
- In `classify`, over subs with a non-None `scattered_light`: compute the session median + σ; a sub is **REJECT** if `scattered_light > median + qa_scatter_reject_sigma*σ` (or `> qa_scatter_absolute` if set), **MARGINAL** if `> median + qa_scatter_marginal_sigma*σ`. Reason: e.g. `"REJECT: scattered light 0.42 > 0.31 (median + 2σ) — likely thin cirrus / bright-star halos"`.
- Fits the existing "any single REJECT trigger ⇒ REJECT; MARGINAL on any metric ⇒ MARGINAL" logic and the "no unexplained rejects" rule. Subs with `scattered_light=None` are simply not scored on this axis (backward compatible — a session with the metric absent behaves exactly as today).
- The `qa-policy` skill gains a line mapping the pattern: **halo ratio / background non-uniformity up, FWHM roughly stable, SNR & star-count only mildly down ⇒ thin cirrus / scattered light** (distinct from focus, and from thick cloud which also collapses star count).

### Fixture

Add `tests/fixtures/hazy.fits`, generated by the existing seeded `make_fixtures.py`: the same clean round stars as `good.fits`, PLUS (a) a smooth large-scale background gradient/pedestal and (b) a broad low-amplitude Gaussian *halo* added around the brightest stars. It must have low FWHM/eccentricity and adequate star count (so it would PASS the old metrics) but a clearly **elevated `scattered_light`** — proving the new metric catches what the others miss.

## Data flow

```
analyze_sub(fits) → detect sources + Background2D → halo ratio (bright stars) + bkg non-uniformity
                  → scattered_light on SubMetrics
classify(session) → session-relative scatter threshold → PASS/MARGINAL/REJECT + reason
qa_session_report → scattered-light appears in metrics + reasons; keep-list excludes hazy subs
```

## Error handling

- < K bright stars, no Background2D, NaN/inf → `scattered_light = None` (never raises; `_finite` guard as elsewhere).
- Non-finite must never reach the report (strict-JSON invariant already enforced by `render_json(allow_nan=False)` + `_finite`).
- Absent metric across a whole session → no scatter verdicts (identical to current behavior).

## Testing

- `analyze_sub(good.fits)` → `scattered_light` is a small finite number (low).
- `analyze_sub(hazy.fits)` → `scattered_light` clearly higher than `good.fits`; FWHM still low + star count adequate (proves it would pass the OLD floors).
- `classify` (hand-built `SubMetrics`): a sub with elevated `scattered_light` vs a session of clean subs → REJECT with a reason naming scattered light + threshold; a session where all `scattered_light` are None → no scatter verdicts (regression: identical to pre-feature).
- Absolute override honored.
- `analyze_sub` on garbage/zero-star → `scattered_light = None`, no raise; `render_json` on a real hazy-included session parses under strict JSON.
- Determinism: seeded fixture; no network/hardware.

## Security / reproducibility

No new deps, no network, no secrets. Deterministic (seeded fixture + pure analysis). Backward compatible (None metric = no behavior change). The metric and its verdict are reason-tagged and auditable like every other Tier-2 figure.
